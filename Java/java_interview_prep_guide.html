You know C++ and Python. Below are the most important syntax differences. Read this once, then write the code yourself.

C++ / Python

// C++ class class Animal { public: string name; Animal(string n) { name = n; } virtual void speak() {} };

Java

// Java class public class Animal { private String name; public Animal(String n) { this.name = n; } public void speak() {} }

C++ vector / map

vector<string> list; list.push_back("hello"); map<string,int> m; m["age"] = 21;

Java Collections

List<String> list = new ArrayList<>(); list.add("hello"); Map<String,Integer> m = new HashMap<>(); m.put("age", 21);
Key Java-specific things to know
  • ✓
    interface vs abstract class — Interface: multiple inheritance, no state. Abstract: single inheritance, can have state. Use interface when you want a contract.
  • ✓
    Checked vs Unchecked exceptions — Checked (IOException) must be declared or caught. Unchecked (NullPointerException) can be ignored.
  • ✓
    String is immutable — Every operation creates a new String. Use StringBuilder for concatenation in loops.
  • ✓
    == vs .equals() — == compares references. .equals() compares content. Always use .equals() for Strings.
  • ✓
    No pointers, no delete — Java has garbage collection. You just stop referencing an object and it gets cleaned up.
  • ✓
    Everything extends Object — All Java classes inherit from Object automatically. toString(), hashCode(), equals() come for free.

Java Core
What is OOP? Explain with an example. ▼
OOP has 4 pillars: Encapsulation (hiding data with private fields + getters/setters), Inheritance (subclass extends superclass), Polymorphism (same method name, different behavior), Abstraction (hiding implementation details via interfaces/abstract classes). Example: Animal is abstract, Dog extends Animal overrides speak() to return "Woof".
What's the difference between interface and abstract class? ▼
Interface: pure contract, no state, a class can implement multiple interfaces. Abstract class: can have fields, constructors, concrete methods, but only single inheritance. Use interface for capabilities (Runnable, Comparable), use abstract class for shared base behavior.
What is multithreading? How do you create a thread? ▼
Multithreading = multiple threads running concurrently. Two ways: (1) extend Thread and override run(), (2) implement Runnable and pass to new Thread(runnable).start(). Prefer Runnable because Java doesn't allow multiple inheritance. Use synchronized keyword to avoid race conditions.
What is exception handling? Checked vs Unchecked? ▼
try block has risky code, catch handles the exception, finally always runs. Checked exceptions (like IOException) must be declared in method signature or caught — compiler enforces this. Unchecked (like NullPointerException) are runtime errors you don't have to declare.
ArrayList vs LinkedList vs HashMap — when to use? ▼
ArrayList: fast random access O(1), slow insert/delete in middle O(n). LinkedList: fast insert/delete O(1), slow access O(n). HashMap: key-value store, average O(1) get/put. Use ArrayList for most lists, HashMap for lookups, LinkedList rarely (queues/deques).
Spring Boot
What is Spring Boot and why use it? ▼
Spring Boot is a framework to build production-ready Java applications quickly. It provides auto-configuration (no XML setup), embedded server (Tomcat built-in), and starter dependencies. Think of it like Express.js but for Java — handles boilerplate so you focus on business logic.
Explain the layers in a Spring Boot app ▼
Three layers: @RestController (handles HTTP requests, like your Express routes), @Service (business logic layer, like your service files in Node), @Repository (database access layer, like your Mongoose models). Each layer talks to the one below it.
What is JPA / Hibernate? ▼
JPA (Java Persistence API) is a specification for ORM — mapping Java objects to database tables. Hibernate is the most popular implementation. Spring Data JPA wraps Hibernate and gives you JpaRepository with built-in methods: save(), findById(), findAll(), deleteById() — no SQL needed for basic operations.
